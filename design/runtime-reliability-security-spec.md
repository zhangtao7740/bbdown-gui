# Runtime / 日志 / 安全 / 测试规范

## 范围决策

必须做：
- 日志系统
- 诊断日志收集
- 安全策略
- 测试矩阵
- breaking change 策略

明确不做：
- 不做 v1 到 v2 数据迁移
- 不做自动更新
- 不做性能指标体系
- 不做新手引导
- 不做 i18n
- 不做拖拽链接或文件

## Breaking Change 策略

本阶段不做历史配置、历史记录和 Cookie 的迁移。

要求：
- 配置版本号直接升到新版本。
- 旧配置不自动读取。
- 旧历史不自动导入。
- 旧 Cookie 不自动复用。
- 首次启动新版本时，如果检测到旧数据，只提示“发现旧版本数据，本版本不自动迁移”，并提供打开旧数据目录。

用户文案：

```text
发现旧版本数据

当前版本调整了配置和历史记录结构，不会自动迁移旧数据。
你可以打开旧数据目录手动备份；新版本会从空配置开始。
```

操作：
- 打开旧数据目录
- 继续使用新版本

## 日志系统

日志级别：

| Level | 用途 |
| --- | --- |
| debug | 开发调试，默认不写入用户可见日志 |
| info | 正常流程，如解析开始、下载开始、合并完成 |
| warn | 可恢复异常，如重试、工具版本读取失败 |
| error | 任务失败、后处理失败、IPC 失败 |

日志来源：
- app
- bbdown
- ffmpeg
- aria2c
- task
- history
- settings

日志存储：
- 运行日志按天滚动。
- 单文件建议上限 5 MB。
- 最多保留 7 天。
- 任务日志和应用日志分开。

日志脱敏：
- 不写入 Cookie。
- 不写入 SESSDATA、bili_jct、DedeUserID。
- URL query 中疑似 token 的字段脱敏。
- 复制诊断信息时再次脱敏。

诊断导出：
- 应用版本
- Electron 版本
- 平台
- BBDown/FFmpeg/aria2c 路径和版本
- 最近 200 行 app 日志
- 当前任务最近 200 行任务日志
- 最近错误堆栈

## 安全策略

Cookie：
- 不在 renderer 内直接展示 Cookie。
- 不复制 Cookie 到诊断信息。
- 删除登录态时明确说明会删除本地凭据文件。

路径：
- 用户输入路径必须规范化。
- 不允许把路径拼接成 shell 字符串执行。
- 使用 `spawn` 参数数组，不使用 shell 拼接。
- 打开目录前确认路径存在。

命令参数：
- 下载 URL/BV 号必须经过格式校验。
- BBDown 参数由白名单选项生成。
- 不允许用户输入任意命令参数直通。
- 文件名模板如果后续支持，必须过滤非法字符。

IPC：
- renderer 只能调用 preload 暴露的 API。
- IPC payload 使用类型校验，P1 可用 `zod`。
- 主进程对路径、任务 id、操作类型再次校验。

外部链接：
- GitHub 或下载链接使用系统浏览器打开。
- 不允许在 Electron WebView 内加载任意外站。

### 功能 smoke test

- 空输入解析。
- 非法链接解析。

- 多分 P 选择。
- 加入队列。

历史页：
- 历史列表刷新。
- 产物展开。
- 打开目录。
- 重命名。
- 重新定位缺失文件。
- 后处理弹窗打开。

设置页：
- 自动检测工具。
- 手动选择工具。
- 保存设置。
- 扫码登录取消。
- 退出登录确认。

关于页：
- 展示运行环境。
- 复制诊断信息。
- 打开项目链接。

### 安全测试

- URL 中包含特殊字符不会进入 shell 拼接。
- 保存目录包含空格、中文、括号可正常使用。
- 文件名包含非法字符时被处理。
- 诊断信息不包含 Cookie。
- 删除历史记录不删除真实文件。

### 响应式测试

- 1280x800 正常三栏。
- 1100x720 右侧状态栏下移或折叠。
- 960x640 不出现核心操作不可见。

## 验收清单

- 日志有级别、来源、滚动和脱敏。
- 诊断信息可复制且不泄露 Cookie。
- breaking change 文案明确，不承诺迁移。
- 测试矩阵覆盖主流程。
- 安全策略覆盖路径、命令参数、IPC 和 Cookie。
